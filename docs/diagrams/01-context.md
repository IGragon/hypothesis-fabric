# C4 Context — Вариант 2 (Hybrid RAG + agent)

```mermaid
C4Context
    title C4 Context: Фабрика гипотез — Hybrid RAG + agent
    Boundary(org, "НИИ / промышленная лаборатория") {
        Person_Ext(1, "Исследователь-материаловед", "Цель + ограничения + фидбек; валидация")
        Person_Ext(2, "Руководитель проекта", "Ранжирование, KPI, экспорт")
    }
    System(sys, "Фабрика гипотез", "Гибрид: детерминированный retrieval-first каркас + LLM в слотах (реранк, генерация, выбор инструментов)")
    System_Ext(3, "База знаний (on-prem)", "Публикации, патенты, отчёты, эксперименты — RU/EN/CN")
    System_Ext(4, "Внешние научные API", "Semantic Scholar, Materials Project, Citrination, NIMS MatNavi")
    System_Ext(5, "Трекеры задач", "Jira / YouTrack (REST API)")
    System_Ext(6, "Сервис идентичности", "SSO + RBAC, локальный")
    Rel(1, sys, "Цель + ограничения + фидбек [R-IN1][R-IN2][R-F14]")
    Rel(sys, 1, "Гипотезы + обоснование + графы [R-OUT1..7][R-F10]")
    Rel(2, sys, "Отчёты/экспорт [R-F12][R-F13]")
    Rel(sys, 3, "Детерминированный ETL/индекс [R-F1][R-F2][R-F3]")
    Rel(sys, 4, "Доп. источники (опционально) [R-K5]")
    Rel(sys, 5, "Задачи на проверку [R-F13]")
    Rel(6, sys, "Аутентификация/авторизация [R-N5]")
```

## Legend
- `Person_Ext` — внешняя роль; `System` — проектируемая система; `System_Ext` —
  внешние системы/границы. `Rel` — связь с тегами `[R-*]`.

## Компоненты и связи

| # | Элемент | Роль / примечание |
|---|---------|-------------------|
| 1 | Исследователь-материаловед | Главный пользователь [R-IN1..3][R-K1]; даёт фидбек [R-F14]. |
| 2 | Руководитель проекта | Ранжирование, KPI-отчёты, экспорт [R-F12][R-F13]. |
| 3 | База знаний (on-prem) | Локальный корпус; индексируется **детерминированно** (в отличие от V1). |
| 4 | Внешние научные API | Доп. источники; опциональны, отключаемы [R-N5]. |
| 5 | Трекеры задач | Jira/YouTrack; приёмник задач [R-F13]. |
| 6 | Сервис идентичности | SSO + RBAC [R-N5]. |
| sys | Фабрика гипотез | Гибрид: retrieval-first каркас + LLM-слоты (см. Container). |

**Отличие от V1**: граница «система ↔ база знаний» — не ленивый парсинг по
запросу агента, а **детерминированный ETL** с предbuilt-индексом и KG. LLM
работает внутри фиксированных слотов, а не управляет всем потоком.
