# Spec: Expert Feedback Loop (M10) — Вариант 2

> Labels → калибровка весов ранжирования. Ссылается на `05-dataflow.md`
> (узел 12), `03-component.md` (#15).

## Роль
Позволить экспертам корректировать, оценивать и «обучать» систему [R-F14][R-A3].
Учитывать подтверждённые/опровергнутые гипотезы в будущих рекомендациях.

## Поток фидбэка
1. Эксперт в UI ставит label (`accepted`/`rejected`/`adjusted`) + комментарий [R-F14].
2. M10 пишет label в Feedback Labels Store (M8) [R-A3].
3. Weighted Ranker (M6) калибрует веса по накопленным labels [R-A3].

## Политика обучения
- Без on-line fine-tuning LLM [ASSUM-7]; только пере-взвешивание.
- Конфликт labels → хранить оба, отмечать; аудит изменений.
- Удаление по запросу (локальный аналог GDPR) [R-N5].

## Failure modes
- Несогласованные labels → отметить конфликт, не усреднять вслепую.
