# Spec: Ingestion & ETL (M1) — Вариант 2

> Предbuilt детерминированный ETL: парсинг, chunking, эмбеддинги, построение KG.
> Ссылается на `02-container.md` (Container 4), `05-dataflow.md` (узел 1).

## Роль
Принять разнородные данные (текст/PDF/Excel/БД), распарсить, разбить на чанки,
вычислить эмбеддинги, извлечь сущности/связи и построить KG — заранее, batch.
Обеспечивает retrieval-first каркас.

## Источники / парсеры

| Формат | Парсер | Требование |
|--------|--------|------------|
| Текст/статьи/патенты | GROBID + SciSpacy | [R-F1] |
| Excel/БД | pandas/JDBC | [R-F2] |
| Метаданные | ISA-Tab/Allotrope-подобная схема | [R-F3] |

## Индекс
- Чанки → мультиязычный эмбеддер → VectorDB [R-N2][ASSUM-2].
- Метаданные (источник/дата/автор/условия) сохраняются с чанком [R-F3].
- idempotent ETL: повторный прогон не дублирует.

## KG-build
- Извлечение сущностей (Material/Parameter/Property/Process) и связей [R-F4].
- LLM-рефинмент/нормализация сущностей (`LLM/Agent` slot) + детерминированный
  provenance [R-F4]; конфликты → ребро `contradicts` [R-N3].
- Хранилище: Neo4j [R-K4].

## Ограничения
- ETL batch latency ≤ 2 ч на 10⁴ док. [ASSUM-1][R-N4].
- FE9: упавшая джоба → retry + dead-letter; индекс на последнем слепке [R-N3].
